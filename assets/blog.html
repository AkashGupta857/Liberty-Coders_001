WebEase UI Library

@Emma_ | July 19, 2024

WebEase UI Library v1.0.0 is here with bug fixes, documentation improvements, and more follow-up enhancements for color modes. Keep reading for the highlights!

Highlights

  • Fixed a breaking change introduced with color modes where it was required to manually import variables-dark.scss when building with Sass. Now, _variables.scss will automatically import _variables-dark.scss.
  • Fixed a regression in the selector engine that wasnâ€™t able to handle multiple IDs anymore.
  • Enhanced accessibility features to ensure better usability for people with disabilities.
  • Improved documentation with more detailed examples and better structure.
  • Optimized performance for faster load times and smoother user experience.
  • Added support for RTL (right-to-left) languages to cater to a broader audience.

New Features

  • Integrated with popular frontend frameworks like React, Angular, and Vue for seamless development.
  • Added a set of new UI components including modals, tooltips, and carousels.
  • Introduced a drag-and-drop builder for faster and more intuitive UI design.
  • Implemented a responsive grid system that adapts to different screen sizes and orientations.
  • Provided extensive theming options to match any branding requirements.
  • Enhanced plugin architecture to allow easy addition of third-party functionalities.

Color Modes

  • Badges now use the .text-bg-* text utilities to be certain that the text is always readable (especially when the customized colors are different in light and dark modes).
  • Used our color-modes.js script to handle the case where the OS is set to light mode and the auto dark mode switch is enabled.
  • Introduced a new dark mode palette that is easier on the eyes during extended use.
  • Added a customizable color scheme feature to allow users to create their own themes.
  • Improved contrast ratios for better readability in all color modes.
  • Included new color utilities for easier and more flexible customization.
